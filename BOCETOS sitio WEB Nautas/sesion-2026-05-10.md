# Sesión 2026-05-10 — Resolución de i18n y Build Bilingüe

## Objetivo
Resolver el problema de páginas en inglés que se generaban con HTML vacío durante el build estático (SSG) de Astro 6.3.1.

## Problema raíz
En Astro 6.3.1, con la configuración:
```js
i18n: {
  defaultLocale: 'es',
  locales: ['es', 'en'],
  routing: {
    prefixDefaultLocale: false,
    redirectToDefaultLocale: false,
  },
  fallback: { en: 'es' },
}
```

Todas las rutas `/en/...` se generaban como tipo "fallback" y producían HTML vacío (`file not created, response body was empty`). Sin el `fallback`, las rutas en inglés directamente no se generaban.

## Solución aplicada
Crear páginas explícitas en inglés dentro del directorio `src/pages/en/`. En Astro 6, las páginas en `src/pages/en/` automáticamente obtienen:
- Prefijo de URL `/en/`
- `Astro.currentLocale = 'en'`
- Tipo de ruta "page" (no "fallback"), que se renderiza correctamente en SSG

### Estructura final de páginas
```
src/pages/
  index.astro                    → / (ES)
  institucional.astro            → /institucional/ (ES)
  proyectos.astro                → /proyectos/ (ES)
  contacto.astro                 → /contacto/ (ES)
  archivo.astro                  → /archivo/ (ES)
  proyectos/[...slug].astro      → /proyectos/[slug]/ (ES, dinámico)
  en/
    index.astro                  → /en/ (EN)
    institucional.astro          → /en/institucional/ (EN)
    proyectos.astro              → /en/proyectos/ (EN)
    contacto.astro               → /en/contacto/ (EN)
    archivo.astro                → /en/archivo/ (EN)
    proyectos/[...slug].astro    → /en/proyectos/[slug]/ (EN, dinámico)
```

Cada página en inglés es idéntica a su contraparte española, con imports ajustados un nivel más de profundidad (`../layouts/` → `../../layouts/`).

## Bug fix adicional: URLs canónicas con doble barra
**Archivo**: `src/layouts/BaseLayout.astro` (línea 22)

**Antes**:
```js
const canonicalPath = currentPath.replace(/^\/(en|es)(\/|$)/, '/$2') || '/';
```

**Después**:
```js
const canonicalPath = currentPath.replace(/^\/(en|es)(\/|$)/, '$2') || '/';
```

El grupo de captura `$2` ya incluye el `/`, por lo que `'/$2'` producía `//proyectos/...` → doble barra en las URLs canónicas.

## Resultado del build
- **24 páginas** generadas correctamente (12 ES + 12 EN)
- Sin errores de "empty body"
- `fallback: { en: 'es' }` permanece en la configuración para páginas sin versión explícita en inglés (ej. `/admin/`)
- Warning menor: `Could not render '/en' from route '/en/' as it conflicts with higher priority route '/en'` — esperado, la página explícita gana la prioridad

## Configuración final (`astro.config.mjs`)
```js
export default defineConfig({
  site: 'https://nautas.org',
  i18n: {
    defaultLocale: 'es',
    locales: ['es', 'en'],
    routing: {
      prefixDefaultLocale: false,
      redirectToDefaultLocale: false,
    },
    fallback: {
      en: 'es',
    },
  },
  vite: {
    plugins: [tailwindcss()],
  },
});
```

## Notas técnicas importantes
- **Astro 6.3.1**: No genera automáticamente rutas para todos los locales en SSG con `prefixDefaultLocale: false`. Se requiere crear directorios explícitos por locale (`src/pages/en/`).
- **Content config**: Usa `src/content.config.ts` (no `src/content/config.ts`) con loaders `glob()` de `astro/loaders`.
- **Rutas dinámicas**: Requieren `getStaticPaths` explícito con `GetStaticPaths` de `astro`.
- **Schema de voces**: Usa `z.union([z.string(), z.object({ name: z.string() })])` para compatibilidad entre Decap CMS (objetos) y el frontend (strings).
